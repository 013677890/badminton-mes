# M1/M2 测试设计与结果

> @author 范家权
> 测试日期：2026-07-13
> 适用里程碑：M1 `barcode` 条码基础能力、M2 `scene` 任务/派工/工序作业
> 代码基线：`PartB` 分支

## 1. 测试目标

本轮测试在不修改生产代码、不修改数据库迁移、不修改 Gradle 配置的前提下，补充 M1/M2
中风险较高但原测试覆盖较薄的边界。测试关注业务结果和跨模块契约，不以调用行数或无效断言
凑用例数量。

主要目标如下：

1. 验证 M1 条码被 M2 扫码使用时，状态、业务归属和使用记录保持一致；
2. 验证 M2 生产任务创建时冻结上游工单快照，并拒绝超量、非法时间和并发状态冲突；
3. 验证 M2 工序作业的操作人权限、扫码记录、关键工序顺序和状态联动；
4. 验证 M2 产品当前状态、状态履历和工序履历经过对象级数据权限检查并正确转换为 VO；
5. 提供不依赖第三方工具的 M1/M2 API 压测入口，并使用失败率与 P95 响应时间作为门禁。

## 4. 压力测试设计

### 4.1 场景

`M1M2ApiPressureRunner` 仅使用 JDK 21 `HttpClient`，支持三个低破坏性读取场景：

| 场景参数 | 请求 | 验证目的 |
| --- | --- | --- |
| `barcode-parse` | `POST /api/barcode/instances/parse` | M1 条码唯一索引读取、解析和统一响应能力 |
| `task-progress` | `GET /api/scene/production_tasks/{id}/progress` | M2 任务与工序进度聚合能力 |
| `my-operations` | `GET /api/scene/operation_jobs/my` | M2 登录人权限过滤和工序列表查询能力 |

压测成功的判定条件为 HTTP 状态码处于 2xx，且响应 JSON 的业务码为 `00000`。运行器统计
吞吐量、平均响应时间、P50、P95、P99、最大响应时间和首个失败样本，并在门禁失败时返回
非零进程退出码。

### 4.2 默认门禁

| 参数 | 默认值 | 含义 |
| --- | ---: | --- |
| `requests` | 1000 | 正式测量请求数 |
| `concurrency` | 20 | 最大并发线程数 |
| `warmup` | 20 | 预热请求数，不计入正式指标 |
| `timeout-seconds` | 10 | 单请求超时 |
| `max-failure-rate` | 0.01 | 最大失败率 1% |
| `max-p95-ms` | 500 | P95 最大 500 ms |

默认值用于开发/测试环境回归，不直接代表生产容量承诺。验收时应根据部署规格分为阶梯并发
（如 10、20、50、100），每档至少稳定运行 3～10 分钟，并同时观察应用、MySQL、Redis
的 CPU、内存、连接池、慢查询和错误日志。

### 4.3 编译与运行

在 `backend-java` 目录执行：

```powershell
$outputDir = "build/stress-test-classes"
New-Item -ItemType Directory -Force $outputDir | Out-Null
javac -encoding UTF-8 -d $outputDir src/stressTest/java/com/badminton/mes/performance/M1M2ApiPressureRunner.java
$env:MES_PRESSURE_TOKEN = "测试环境登录 Token"
```

M1 条码解析：

```powershell
java -cp $outputDir com.badminton.mes.performance.M1M2ApiPressureRunner `
  --scenario barcode-parse `
  --base-url http://127.0.0.1:8080 `
  --barcode BC202607130001 `
  --requests 1000 `
  --concurrency 20
```

M2 任务进度：

```powershell
java -cp $outputDir com.badminton.mes.performance.M1M2ApiPressureRunner `
  --scenario task-progress `
  --base-url http://127.0.0.1:8080 `
  --task-id 1 `
  --requests 1000 `
  --concurrency 20
```

M2 我的工序：

```powershell
java -cp $outputDir com.badminton.mes.performance.M1M2ApiPressureRunner `
  --scenario my-operations `
  --base-url http://127.0.0.1:8080 `
  --requests 1000 `
  --concurrency 20
```

必须使用隔离测试环境和专用账号；条码、任务必须存在且在账号数据权限范围内。禁止对生产环境
直接使用本文默认压力参数。
