# 电脑重启与前后端代码修改后的启动、构建和 Docker 重构命令

本文档适用于当前羽毛球 MES 项目的 Windows PowerShell 开发环境，项目根目录为：

```text
D:\SoftwareAnalysis\badmintonProject\badminton-mes
```

当前运行结构：

```text
微信小程序 / Vue 管理端
        ↓ HTTP API
Docker: mes-backend（Spring Boot）
        ↓
Docker: mes-mysql + mes-redis
```

当前 `docker-compose.yml` 只负责以下三个服务：

```text
mes-backend
mes-mysql
mes-redis
```

微信小程序和 `frontend-web` 当前不在 Docker 中运行，因此只修改前端代码时通常不需要重建 Docker。

> 所有命令都尽量写成可单独执行的完整命令，不使用 PowerShell 反引号续行。请在 PowerShell 中逐行执行。

## 一、重要注意事项

1. 不要关闭 Clash。微信官方接口依赖 Clash 桥接端口访问。
2. 不要随意执行 `docker compose down -v`，该命令会删除 MySQL 和 Redis 数据卷。
3. 不要把真实 AppSecret、MySQL 密码或 Redis 敏感配置提交到 Git。
4. `.env` 是本机私密配置文件，已经由 `.gitignore` 忽略。
5. 已经执行过的 Flyway 版本迁移不要直接修改，应新增下一条迁移。
6. 微信开发者工具使用 `127.0.0.1:8080`；真机访问使用 WLAN 地址的 `18080` 端口。

## 二、最常用命令速查表

| 场景 | 是否重建 Docker | 推荐操作 |
| --- | --- | --- |
| 电脑重启后恢复开发环境 | 通常不重建 | 启动 Clash 桥接，然后执行 `docker compose up -d` |
| 只修改微信小程序 TS/WXML/WXSS | 否 | `npm run typecheck`、`npm run build`、开发者工具重新编译 |
| 只修改 Vue 管理端 | 否 | `npm run build` 或 `npm run dev` |
| 修改 Java 后端代码 | 是 | `docker compose up -d --build backend` |
| 修改后端 `application.yml` | 建议重建 | `docker compose up -d --build --force-recreate backend` |
| 修改 `.env` 中后端环境变量 | 不一定构建，但必须重建容器 | `docker compose up -d --force-recreate backend` |
| 修改 `Dockerfile`、`build.gradle` 或依赖 | 是，建议无缓存 | `docker compose build --no-cache backend` |
| 新增 Flyway SQL | 是 | 备份数据库，然后重建并启动 `backend` |
| 只修改小程序 API 地址 | 否 | 修改配置、执行小程序构建、开发者工具重新编译 |
| 修改 `docker-compose.yml` | 视修改范围 | 使用 `--force-recreate` 重新创建受影响服务 |

## 三、电脑重启后恢复项目环境

### 3.1 进入项目根目录

```powershell
cd D:\SoftwareAnalysis\badmintonProject\badminton-mes
```

后续 Docker 和脚本命令默认都在该目录执行。

### 3.2 确认 Clash 正常运行

```powershell
Get-Service clash_verge_service
```

```powershell
Get-Process verge-mihomo
```

预期看到：

```text
clash_verge_service  Running
verge-mihomo         正在运行
```

检查 Clash 本地混合代理端口：

```powershell
netstat -ano | findstr ":7897"
```

预期存在：

```text
127.0.0.1:7897 LISTENING
```

如果端口不是 `7897`，启动微信桥接时需要通过 `-ClashPort` 指定实际端口，同时更新 `.env` 中的代理端口。

### 3.3 启动微信官方接口的 Clash 桥接

```powershell
powershell.exe -NoProfile -ExecutionPolicy Bypass -File .\scripts\start-wechat-clash-relay.ps1 -ForceRestart
```

该脚本建立以下链路：

```text
mes-backend
→ host.docker.internal:17897
→ 127.0.0.1:7897
→ Clash
→ api.weixin.qq.com
```

检查桥接端口：

```powershell
netstat -ano | findstr ":17897"
```

预期存在：

```text
0.0.0.0:17897 LISTENING
```

如果 Clash 使用的实际端口不是 `7897`，例如是 `7890`：

```powershell
powershell.exe -NoProfile -ExecutionPolicy Bypass -File .\scripts\start-wechat-clash-relay.ps1 -ClashPort 7890 -ForceRestart
```

并修改本机 `.env`：

```env
MES_WECHAT_PROXY_HOST=host.docker.internal
MES_WECHAT_PROXY_PORT=17897
```

`MES_WECHAT_PROXY_PORT` 是 Docker 后端访问的桥接端口，不是 Clash 自己的端口。

### 3.4 启动 Docker 服务

确认 Docker Desktop 已经启动，然后执行：

```powershell
docker compose up -d
```

该命令不会主动重新构建镜像，只会启动或恢复已有容器。

查看容器：

```powershell
docker ps
```

预期至少存在：

```text
mes-backend
mes-mysql
mes-redis
```

查看 Compose 服务状态：

```powershell
docker compose ps
```

### 3.5 验证 MySQL、Redis 和后端

验证 MySQL：

```powershell
docker exec mes-mysql sh -c 'mysqladmin ping -h 127.0.0.1 -uroot -p"$MYSQL_ROOT_PASSWORD" --silent'
```

预期：

```text
mysqld is alive
```

验证 Redis：

```powershell
docker exec mes-redis redis-cli ping
```

预期：

```text
PONG
```

验证后端公共接口：

```powershell
Invoke-WebRequest -UseBasicParsing http://127.0.0.1:8080/api/system/auth/registration_roles
```

如果返回 HTTP 200，说明微信开发者工具可以访问后端。

查看后端最近日志：

```powershell

```

持续查看后端日志：

```powershell

```

退出持续日志查看使用：

```text

```

### 3.6 需要真机访问时启动 WLAN 代理

微信开发者工具在电脑上运行时直接访问：

```text
http://127.0.0.1:8080
```

只有真机调试或手机访问局域网后端时，才需要启动 `18080` WLAN 代理。

先查看 WLAN IPv4：

```powershell
ipconfig
```

也可以自动读取：

```powershell
$wlanIp = Get-NetIPAddress -AddressFamily IPv4 -InterfaceAlias 'WLAN' | Where-Object { $_.IPAddress -notlike '169.254*' } | Select-Object -First 1 -ExpandProperty IPAddress
```

查看读取结果：

```powershell
$wlanIp
```

以管理员权限配置防火墙并启动 WLAN 代理：

```powershell
powershell.exe -NoProfile -ExecutionPolicy Bypass -File .\scripts\setup-miniapp-firewall.ps1 -WlanAddress $wlanIp -ListenPort 18080 -UpstreamPort 8080
```

验证手机访问入口：

```powershell
Invoke-WebRequest -UseBasicParsing ("http://{0}:18080/api/system/auth/registration_roles" -f $wlanIp)
```

检查端口：

```powershell
netstat -ano | findstr ":18080"
```

如果 WLAN IPv4 改变，还需要同步修改：

```text
miniapp-wechat/miniprogram/services/config.ts
```

将：

```typescript
DEFAULT_API_BASE_URL = 'http://旧IP:18080'
```

修改为：

```typescript
DEFAULT_API_BASE_URL = 'http://当前WLAN-IP:18080'
```

然后重新构建微信小程序。


### 5.4 无缓存重建后端

适用于以下情况：

- 修改了 `backend-java/Dockerfile`；
- 修改了 `build.gradle` 依赖；
- Docker 构建缓存异常；
- 明明修改了代码，但镜像仍像是旧版本。

执行：

```powershell
docker compose build --no-cache backend
```

```powershell
docker compose up -d backend
```

也可以强制重新创建：

```powershell
docker compose up -d --force-recreate backend
```

### 5.5 只重启后端，不重新构建

如果没有修改代码和镜像，只需要重新启动：

```powershell
docker compose restart backend
```

或：

```powershell
docker restart mes-backend
```

## 九、修改 `.env` 或 Docker Compose 后的命令

### 9.1 修改普通后端环境变量

例如修改：

```text
MES_WECHAT_MINI_APP_ID
MES_WECHAT_MINI_APP_SECRET
MES_WECHAT_CODE_ENV_VERSION
MES_WECHAT_PROXY_HOST
MES_WECHAT_PROXY_PORT
MES_CORS_ALLOWED_ORIGINS
```

环境变量变化不会通过普通 `docker compose restart` 自动注入，必须重新创建容器：

```powershell
docker compose up -d --force-recreate backend
```

如果后端代码也变了：

```powershell
docker compose up -d --build --force-recreate backend
```

### 9.2 修改 `docker-compose.yml`

只修改后端环境变量、端口或启动参数：

```powershell
docker compose up -d --force-recreate backend
```

修改了后端构建上下文或 Dockerfile：

```powershell
docker compose up -d --build --force-recreate backend
```

修改了 MySQL、Redis 和后端的 Compose 配置：

```powershell
docker compose up -d --force-recreate mysql redis backend
```

该命令不会删除命名数据卷，但会短暂重启数据库和 Redis。

### 9.3 MySQL 密码变化的重要说明

MySQL 镜像的以下环境变量主要在数据卷第一次初始化时生效：

```text
MYSQL_ROOT_PASSWORD
MYSQL_USER
MYSQL_PASSWORD
```

如果已有 `mysql-data` 数据卷，仅修改 `.env` 并重建容器不一定会自动修改数据库内部账号密码。

不要为了同步密码直接执行：

```text
docker compose down -v
```

应通过 MySQL `ALTER USER` 修改账号，或者在明确备份并确认需要重建开发库后再执行数据库重建流程。

## 十、修改 Flyway SQL 后的命令

### 10.1 先备份数据库

确保备份目录存在：

```powershell
New-Item -ItemType Directory -Force .\database-backups
```

先生成备份文件名：

```powershell
$backupName = "badminton_mes_$(Get-Date -Format yyyyMMdd_HHmmss).sql"
```

在 MySQL 容器内部生成备份，避免 Windows PowerShell 重定向改变 SQL 文件编码：

```powershell
docker exec mes-mysql sh -c 'mysqldump -uroot -p"$MYSQL_ROOT_PASSWORD" --single-transaction --routines --triggers badminton_mes > /tmp/badminton_mes_backup.sql'
```

复制到项目备份目录：

```powershell
docker cp mes-mysql:/tmp/badminton_mes_backup.sql ".\database-backups\$backupName"
```

删除容器内临时文件：

```powershell
docker exec mes-mysql rm -f /tmp/badminton_mes_backup.sql
```

查看最新备份：

```powershell
Get-ChildItem .\database-backups\*.sql | Sort-Object LastWriteTime -Descending | Select-Object -First 5 Name,Length,LastWriteTime
```

### 10.2 Flyway 迁移规则

正式结构迁移目录：

```text
backend-java/src/main/resources/db/migration
```

开发演示数据目录：

```text
backend-java/src/main/resources/db/dev-migration
```

已经执行过的版本脚本不要直接修改。例如已经执行了：

```text
V2026071517__create_wechat_user_binding.sql
```

后续变更应新增更高版本的迁移，而不是修改旧文件。

### 10.3 执行新迁移

新 Flyway 文件加入后：

```powershell
docker compose up -d --build backend
```

查看 Flyway 启动日志：

```powershell
docker logs --tail 500 mes-backend 2>&1 | Select-String -Pattern "Flyway|Migrat|Successfully applied|Schema"
```

查看迁移历史表：

```powershell
docker exec mes-mysql sh -c 'mysql -u"$MYSQL_USER" -p"$MYSQL_PASSWORD" badminton_mes -e "SELECT installed_rank, version, description, success FROM flyway_schema_history ORDER BY installed_rank;"'
```

查看失败迁移：

```powershell
docker exec mes-mysql sh -c 'mysql -u"$MYSQL_USER" -p"$MYSQL_PASSWORD" badminton_mes -e "SELECT installed_rank, version, description, success FROM flyway_schema_history WHERE success = 0 ORDER BY installed_rank;"'
```

不要在没有确认原因和备份的情况下直接运行 Flyway repair。

预期看到类似：

```text
微信 HTTP 代理已启用 proxyHost: host.docker.internal, proxyPort: 17897
```

### 13.9 微信码生成失败时查看错误

```powershell
docker logs --since 10m mes-backend 2>&1 | Select-String -Pattern "微信接口调用失败|微信小程序码请求|微信 access_token|C0001"
```

常见错误：

| 现象 | 原因 | 处理方式 |
| --- | --- | --- |
| `C0001` 且代理端口未监听 | 微信 Clash 桥接没有启动 | 执行 `start-wechat-clash-relay.ps1` |
| `40001 invalid credential` | Redis 中 token 失效或被更新 | 后端会自动清除并重新获取；再次生成即可 |
| 开发版小程序码无法扫码 | 扫码微信不是项目或体验成员 | 在微信公众平台添加成员，或上传体验版 |
| 真机请求提示不合法域名 | 普通微信环境限制 HTTP/局域网地址 | 使用真机调试，或配置公网 HTTPS 合法域名 |
| 小程序仍显示旧页面 | 开发者工具仍使用旧 JS | 执行 `npm run build` 并清除文件缓存后编译 |

## 十五、明确禁止直接执行的高风险命令

除非已经完成数据库备份并明确要清空开发环境，否则不要执行：

```text
docker compose down -v
docker volume rm badminton-mes_mysql-data
docker volume rm badminton-mes_redis-data
git reset --hard
```

这些命令可能删除数据库、Redis 数据或未提交代码。
