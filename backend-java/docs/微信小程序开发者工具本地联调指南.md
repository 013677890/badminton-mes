# 微信小程序开发者工具本地联调指南

本文适用于当前羽毛球 MES 项目的微信小程序和 Spring Boot 后端本地联调。

小程序工程目录：

```text
D:\SoftwareAnalysis\badmintonProject\badminton-mes\miniapp-wechat
```

后端工程目录：

```text
D:\SoftwareAnalysis\badmintonProject\badminton-mes\backend-java
```

当前小程序 AppID：

```text
wx472e037adad346cf
```

## 一、准备环境

建议使用以下工具：

- 微信开发者工具；
- JDK 21；
- Docker Desktop；
- Node.js 和 npm；
- 可访问本项目的 MySQL、Redis 环境。

后端当前使用 Gradle Wrapper，不需要全局安装 Gradle。

## 二、启动 MySQL 和 Redis

在项目根目录执行：

```powershell
cd D:\SoftwareAnalysis\badmintonProject\badminton-mes
docker compose up -d mysql redis
```

如果不确定服务名称，可以先查看：

```powershell
docker compose config --services
```

查看容器状态：

```powershell
docker compose ps
```

必须确认 MySQL 和 Redis 均处于运行状态。

如果使用的是远程 MySQL/Redis，不需要启动 Docker，但要保证 `application-dev.yml` 或本地环境变量中的连接地址、端口、用户名和密码正确。

## 三、配置微信 AppID 和 AppSecret

后端微信登录需要 AppID 和 AppSecret。真实密钥不要提交到 Git。

AppSecret 不能从当前代码仓库、本机端口或微信开发者工具中推导出来，它是微信平台为小程序分配的服务端密钥，必须在微信公众平台获取。

获取步骤：

1. 打开微信公众平台：<https://mp.weixin.qq.com/>。
2. 使用具有该小程序管理权限的管理员账号登录。
3. 进入“开发”或“开发管理”。
4. 打开“开发设置”。
5. 找到“开发者 ID”区域。
6. 确认 AppID 为 `wx472e037adad346cf`。
7. 在“AppSecret（小程序密钥）”处点击“生成”或“重置”。
8. 完成管理员扫码或身份验证后复制 AppSecret。

AppSecret 通常只在生成或重置时显示一次。如果已经遗失，需要在该页面重新生成；重置后旧 AppSecret 会失效，所有使用旧密钥的后端环境都必须同步更新。

在项目根目录创建或修改 `.env`：

```text
MES_WECHAT_MINI_APP_ID=wx472e037adad346cf
MES_WECHAT_MINI_APP_SECRET=填写微信公众平台中的AppSecret
```

如果使用 Docker Compose，根目录 `.env` 会由 Compose 自动读取。重新构建并启动后端：

```powershell
docker compose up -d --build backend
```

如果直接通过 Gradle 在宿主机运行后端，则必须在当前 PowerShell 会话中设置环境变量；Spring Boot 不会默认自动读取根目录 `.env`：

```powershell
$env:MES_WECHAT_MINI_APP_ID="wx472e037adad346cf"
$env:MES_WECHAT_MINI_APP_SECRET="真实AppSecret"
cd D:\SoftwareAnalysis\badmintonProject\badminton-mes\backend-java
.\gradlew.bat bootRun
```

可以在不打印密钥内容的前提下检查变量是否已经设置：

```powershell
if ([string]::IsNullOrWhiteSpace($env:MES_WECHAT_MINI_APP_SECRET)) {
    Write-Host "MES_WECHAT_MINI_APP_SECRET 尚未设置"
} else {
    Write-Host "MES_WECHAT_MINI_APP_SECRET 已设置"
}
```

后端配置文件已经预留以下配置：

```yaml
mes:
  wechat:
    mini-app:
      app-id: ${MES_WECHAT_MINI_APP_ID:}
      app-secret: ${MES_WECHAT_MINI_APP_SECRET:}
      code-session-url: ${MES_WECHAT_CODE_SESSION_URL:https://api.weixin.qq.com/sns/jscode2session}
      connect-timeout: ${MES_WECHAT_CONNECT_TIMEOUT:3s}
      read-timeout: ${MES_WECHAT_READ_TIMEOUT:5s}
      bind-ticket-ttl: ${MES_WECHAT_BIND_TICKET_TTL:5m}
```

小程序工程的 AppID 已经写入：

```text
miniapp-wechat/project.config.json
```

对应配置为：

```json
{
  "appid": "wx472e037adad346cf"
}
```

## 四、修改小程序后端地址

打开：

```text
miniapp-wechat/miniprogram/services/http.ts
```

当前仓库已配置为：

```ts
const BASE_URL = 'http://127.0.0.1:8080'
```

这个地址已经通过仓库配置确认：

- `application.yml` 默认使用 `server.port=8080`；
- `docker-compose.yml` 使用 `8080:8080` 端口映射；
- 微信开发者工具与后端运行在同一台电脑时，可通过 `127.0.0.1:8080` 访问。

如果后端运行在局域网中的另一台电脑，例如局域网 IP 为 `192.168.1.100`，则改为：

```ts
const BASE_URL = 'http://192.168.1.100:8080'
```

使用局域网地址时，需要确保：

- 开发者工具所在电脑能够访问该地址；
- Windows 防火墙允许 8080 端口；
- Spring Boot 实际监听端口为 8080；
- 局域网设备和后端服务器处于同一网络。

## 五、启动 Spring Boot 后端

打开新的 PowerShell 窗口：

```powershell
cd D:\SoftwareAnalysis\badmintonProject\badminton-mes\backend-java
.\gradlew.bat bootRun
```

如需显式使用开发环境：

```powershell
$env:SPRING_PROFILES_ACTIVE="dev"
.\gradlew.bat bootRun
```

启动成功后，日志中应出现类似信息：

```text
Started ...
Tomcat started on port 8080
```

可以使用以下命令检查端口是否有服务响应：

```powershell
Invoke-WebRequest http://127.0.0.1:8080
```

如果返回 HTTP 404，但请求确实到达 Spring Boot，说明服务通常已经启动；根路径没有业务接口并不代表后端异常。

## 六、执行数据库迁移

微信绑定表通过 Flyway 自动迁移，脚本位于：

```text
backend-java/src/main/resources/db/migration/V2026071517__create_wechat_user_binding.sql
```

正常启动后，Flyway 会自动创建：

```text
sys_wechat_user_binding
```

如果启动失败，重点检查：

- MySQL 是否运行；
- 数据库名称是否正确；
- 数据库用户名和密码是否正确；
- Flyway 历史表是否存在冲突版本；
- MySQL 是否为 8.0/8.4 兼容版本。

## 八、关闭本地开发域名校验

在微信开发者工具中打开：

```text
详情 → 本地设置
```

勾选：

```text
不校验合法域名、web-view（业务域名）、TLS 版本以及 HTTPS 证书
```

本设置只用于本地开发。

正式发布时不能依赖此选项，必须：

- 使用 HTTPS 请求地址；
- 使用 WSS WebSocket 地址；
- 在微信公众平台配置 request 合法域名；
- 在微信公众平台配置 socket 合法域名；
- 使用与小程序主体匹配的正式域名和证书。

## 十、微信登录联调流程

完整登录流程如下：

```text
小程序调用 wx.login()
        ↓
POST /api/system/mini_app/auth/login
        ↓
后端调用微信 code2Session
        ↓
已绑定：返回 Bearer Token
未绑定：返回 bindTicket
        ↓
用户输入 MES 工号和密码
        ↓
POST /api/system/mini_app/auth/bind
        ↓
保存微信身份和 MES 用户绑定关系
        ↓
返回 Bearer Token
```

登录成功后，小程序会将 token 保存到本地缓存，后续请求自动添加：

```http
Authorization: Bearer {token}
```

## 十二、常见问题

### 1. 请求失败或无法连接后端

检查：

- `http.ts` 中的 `BASE_URL` 是否正确；
- Spring Boot 是否运行在 8080 端口；
- 开发者工具是否关闭域名校验；
- Windows 防火墙是否拦截 8080；
- 后端是否能够连接 MySQL 和 Redis。

### 2. 提示微信配置缺失

检查 `.env` 中是否存在：

```text
MES_WECHAT_MINI_APP_ID=wx472e037adad346cf
MES_WECHAT_MINI_APP_SECRET=真实AppSecret
```

修改环境变量后需要重新启动 Spring Boot。

### 3. 提示微信登录凭证失效

常见原因：

- 小程序 AppID 和后端 AppID 不一致；
- `code` 已过期或重复使用；
- AppSecret 错误；
- 后端访问微信接口失败；
- 微信开发者工具没有使用当前项目对应的 AppID。

### 4. 登录成功但看板没有数据

检查当前 MES 用户：

- 用户是否启用；
- 是否分配角色；
- 是否配置车间或产线范围；
- 数据库是否存在生产任务、报工和质量数据；
- 当前查询条件是否超出用户数据权限。

### 5. 页面编译失败

在小程序目录执行：

```powershell
cd D:\SoftwareAnalysis\badmintonProject\badminton-mes\miniapp-wechat
npm install
npm run build
npm run typecheck
```

`npm run build` 会把 `miniprogram` 下的 TypeScript 源文件编译为微信开发者工具实际加载的 JavaScript 文件，例如：

```text
miniprogram/app.ts                  -> miniprogram/app.js
miniprogram/pages/login/login.ts    -> miniprogram/pages/login/login.js
miniprogram/services/http.ts        -> miniprogram/services/http.js
```

修改 `.ts` 文件后需要重新执行 `npm run build`，再回到微信开发者工具点击“编译”。如果 TypeScript 检查失败，先修复类型错误。
