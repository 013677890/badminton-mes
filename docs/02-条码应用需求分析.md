# 条码应用需求分析

条码应用模块用于统一管理 MES 中使用的条码类型、条码规则、标签模板、应用规则和条码生成。条码支持唯一码和批次码两种，课程设计中以批次码为追溯主线，用于支撑羽毛球生产批次追溯。

## 1. 条码类型

### 功能概述

条码类型用于定义系统中可使用的条码分类，支持产品码、内外箱码、中箱码、栈板码和材料码。羽毛球 MES 中，产品码用于成品批次或单件追溯，材料码用于球头、羽片、胶水、线圈等关键物料批次，内外箱码、中箱码和栈板码用于装箱和物流环节记录包装绑定关系。

### 业务流程

1. 系统管理员维护条码类型。
2. 业务人员在条码应用规则中选择条码类型。
3. 系统根据条码类型控制条码使用场景。
4. 现场扫码时，系统根据条码类型识别业务对象。

### 参与角色

- 系统管理员：维护条码类型。
- 工艺工程师：确认条码在工序中的使用场景。
- 操作工：在现场扫码使用条码。

### 输入输出

- 输入：条码类型编码、类型名称、适用对象、启用状态、备注。
- 输出：条码类型列表、条码类型详情、可用条码类型选项。

### 主要数据

- 条码类型表：记录类型编码、名称、业务对象和状态。
- 条码使用场景表：记录条码类型与业务功能的关系。

### 规则与异常

- 条码类型编码必须唯一。
- 已被条码规则使用的条码类型不允许直接删除。
- 停用条码类型后，不允许新建相关条码应用规则。

### 接口需求

- 向条码规则、条码模板、现场扫码和产品追溯提供条码类型数据。

### 验收标准

- 能新增、修改、启用、停用和查询条码类型。
- 条码类型停用后，不再出现在新建规则的可选项中。

## 2. 条码规则

### 功能概述

条码规则用于定义条码生成方式，支持常量、日期、产品编码、产线编码、批次流水号等组合。课程设计中建议采用“产品编码 + 日期 + 批次流水号”的批次码规则。

### 业务流程

1. 系统管理员或业务管理员创建条码规则。
2. 设置规则组成项、长度、日期格式和流水号位数。
3. 条码应用规则绑定具体物料或产品。
4. 条码生成时按规则自动生成批次码。

### 参与角色

- 系统管理员：维护条码规则。
- PMC 计划员：在生产批次中使用条码规则。
- 操作工：扫码识别生产批次。

### 输入输出

- 输入：规则编码、规则名称、组成项、日期格式、流水号长度、适用条码类型。
- 输出：规则预览、生成示例、规则列表。

### 主要数据

- 条码规则表：记录规则基本信息。
- 条码规则明细：记录常量、变量、日期和流水号组成。
- 条码流水号记录：记录当前流水号和重置周期。

### 规则与异常

- 条码规则生成结果必须唯一。
- 流水号达到最大值时应提示规则容量不足。
- 规则修改后，只影响新生成条码，不影响历史条码。
- 规则预览应能提前发现长度或变量缺失问题。

### 接口需求

- 向条码生成和生产任务批次创建提供编码能力。
- 向产品追溯提供条码解析依据。

### 验收标准

- 能配置常量、日期、变量和流水号。
- 能预览条码生成效果。
- 批量生成条码时不出现重复码。

## 3. 条码模板

### 功能概述

条码模板用于定义标签打印格式和显示参数，包括条码、二维码、产品名称、批次号、生产日期、工单号等字段。

### 业务流程

1. 管理员创建标签模板。
2. 设置模板尺寸、字段位置、字体和条码区域。
3. 将模板绑定到条码应用规则。
4. 条码生成后按模板打印标签。

### 参与角色

- 系统管理员：维护模板。
- 车间主管：确认现场标签内容是否满足使用。
- 操作工：打印并粘贴标签。

### 输入输出

- 输入：模板名称、纸张尺寸、字段布局、条码字段、打印份数规则。
- 输出：标签模板、标签预览、打印内容。

### 主要数据

- 条码模板表：记录模板名称、尺寸、状态。
- 模板字段表：记录字段名称、数据来源、显示位置。
- 模板版本记录：记录模板变更历史。

### 规则与异常

- 模板必须包含条码值或二维码值。
- 已绑定应用规则的模板修改后应形成版本记录。
- 打印字段为空时，应提示不能打印或显示默认占位信息。

### 接口需求

- 向条码生成和条码打印功能提供模板。
- 从生产工单、产品档案和批次信息读取标签字段。

### 验收标准

- 能新增、编辑、预览和停用条码模板。
- 打印预览能显示条码、产品、批次和日期信息。

## 4. 条码应用规则

### 功能概述

条码应用规则用于将产品、物料、条码类型、条码规则和标签模板进行组合，明确某类业务对象应按什么规则生成和打印条码。条码支持唯一码和批次码两种：唯一码一物一码，用于单件追溯；批次码一批一码，用于批次追溯。条码来源分为条码规则生成、传入值生成和外部导入三种。

### 业务流程

1. 管理员选择适用产品或物料。
2. 绑定条码类型、条码规则和标签模板。
3. 设置条码来源，包括规则生成、传入值生成或外部导入。
4. 生产任务创建或现场作业时按应用规则生成条码。

### 参与角色

- 系统管理员：维护规则。
- 工艺工程师：确认规则与工艺场景匹配。
- PMC 计划员：在生产任务中使用规则。

### 输入输出

- 输入：产品或物料编码、条码类型、条码模式（唯一码或批次码）、条码规则、标签模板、条码来源、是否启用。
- 输出：条码应用规则列表、规则详情、可用规则选项。

### 主要数据

- 条码应用规则表：记录规则与产品、物料的绑定关系。
- 应用规则版本：记录规则调整历史。
- 条码来源配置：记录规则生成、传入值或外部导入方式。

### 规则与异常

- 同一产品同一条码类型只能启用一条默认规则。
- 条码规则、模板或条码类型停用后，应用规则不能继续启用。
- 外部导入条码必须校验唯一性和格式合法性。

### 接口需求

- 向条码生成、生产任务、现场扫码和产品追溯提供应用规则。

### 验收标准

- 能将产品与条码规则、模板进行绑定。
- 能区分唯一码和批次码两种应用方式。
- 能按不同条码来源生成或导入条码。
- 规则停用后不能继续用于新业务。

## 5. 条码生成

### 功能概述

条码生成功能用于按照条码应用规则生成条码，按应用规则的条码模式生成唯一码或批次码，支持批量生成、外部导入、打印和作废。该功能为产品追溯提供基础数据。

### 业务流程

1. 用户选择生产工单或生产任务单。
2. 系统读取条码应用规则，生成对应数量的批次码。
3. 用户预览条码结果并执行打印。
4. 现场扫码时，系统将批次码关联到生产任务和工序报工。
5. 若条码打印错误或业务取消，可作废未使用条码。

### 参与角色

- PMC 计划员：为生产任务生成批次码。
- 产线班组长：打印和分发条码。
- 操作工：扫码使用条码。
- 质检员：按批次码记录检验结果。

### 输入输出

- 输入：生产任务单、产品编码、生成数量、条码应用规则、打印模板。
- 输出：条码清单、条码标签、打印记录、条码状态。

### 主要数据

- 条码主表：记录条码值、条码类型、业务对象和状态。
- 条码打印记录：记录打印人、打印时间和打印次数。
- 条码使用记录：记录扫码时间、工序、人员和设备。

### 规则与异常

- 条码值必须唯一。
- 已使用条码不能作废。
- 重复打印需要记录打印次数和原因。
- 导入条码必须校验格式和重复性。

### 接口需求

- 向现场管理提供扫码识别能力。
- 向质量管理和产品追溯提供批次码关联。
- 可与外部条码打印设备或打印服务对接。

### 验收标准

- 能按任务批量生成批次码。
- 能打印条码标签并记录打印历史。
- 扫码后能识别产品批次和关联任务。
- 重复码、无效码和已作废码能被拦截。
