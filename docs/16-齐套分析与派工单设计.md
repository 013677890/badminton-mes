## 1. 定位与范围

对应 [01-生产订单需求分析.md](01-生产订单需求分析.md) §2 齐套分析与欠料看板、§3 派工单，属 A 工作包 `module.production`。生产工单（§1）已落地，本篇补齐生产订单模块剩余两块，分两轮实施：Round A 齐套分析（派工建议需读工单 `kit_status` 冗余字段，齐套先行）、Round B 派工单。


## 2. 齐套分析（Round A）

### 2.1 计算规则

对应业务 SQL 1.3，改为应用层计算（明细规模小、便于单测），同一事务内完成：

1. 锁定工单行（`SELECT ... FOR UPDATE`，与派工共用一把锁，防并发重算残留双份结果、防"边派工边重算"）；校验工单状态 ∈ {已下达, 生产中}。
2. 读取 `prod_work_order_material` 需求明细，为空则报错（须先下达生成物料需求）。
3. 批量读取 `material_stock` 快照，逐物料计算：
   - 剩余需求 = require_quantity − issued_quantity
   - 可用 = max(available − locked − checking, 0)，无库存记录按 0
   - 欠料 = max(剩余需求 − 可用, 0)
   - 行状态：剩余需求 ≤ 可用 → 1 齐套；可用 > 0 → 2 部分齐套；否则 → 3 欠料
4. 软删旧分析结果，插入新结果（analysis_time = 当前时间）。
5. 工单级齐套状态 = MAX(行状态)（欠料 > 0 不得标记齐套），冗余回写 `prod_work_order.kit_status` 并清工单详情缓存。

### 2.2 欠料看板

- 看板汇总：对 `kit_analysis` 未删除且 shortage_quantity > 0 的行，按物料 GROUP BY 汇总欠料总量、影响工单数、在途数量，欠料量降序；物料编码/名称批量回查补齐。
- 下钻：按物料查影响的工单明细（工单号、需求、欠料）。
- 预计到料：来自欠料处理记录的 expected_arrival_date（看板取该物料未解决记录的最近一条）。

## 3. 派工单（Round B）

### 3.1 防超派（业务 SQL 1.4）

创建派工单事务：
1. `SELECT ... FOR UPDATE` 锁工单行，校验状态 ∈ {已下达, 生产中}。
2. 应用层校验：本次派工数 ≤ FLOOR(plan_quantity × (1 + over_ratio/100)) − dispatched_quantity。
3. 校验产线/班次启用、planStartTime < planEndTime、工厂日历该车间该日为工作日（无记录默认工作日）。
4. 产能校验：该产线该日该班次已排量（排除已取消）+ 本次 ≤ standard_capacity ÷ 当日启用班次数（capacity 为 NULL 跳过）。跨工单同线并发的轻微超产能竞态本期接受（严格需锁产线行，延后）。
5. Redis 序列生成派工单号（前缀 DP + 日期 + 流水），INSERT（状态 0 待审核）。
6. `UPDATE prod_work_order SET dispatched_quantity = dispatched_quantity + n WHERE dispatched_quantity + n <= FLOOR(...)` 兜底，影响行数 0 → 抛错回滚。
7. 写排产调整日志，清工单缓存。欠料工单不阻断创建（需求：未齐套可生成待确认派工），响应透出 kitStatus 供前端警示。

### 3.2 状态机

`0待审核 →(审核)→ 1已审核 →(下发)→ 2已下发`；`0/1/2 →(取消)→ 5已取消`（锁工单后回退 dispatched_quantity）；`3执行中/4已完成` 由现场模块回写，本期仅保留状态值。CAS UPDATE 影响行数判定，同工单状态机风格。

修改规则：状态 0 直接改（重新走数量/产能校验，差额调整 dispatched_quantity）；状态 2 修改必填 adjust_reason 并记录调整前后快照；状态 1 需先取消审核？——简化：状态 1 同状态 0 允许修改；3/4/5 不允许。

### 3.3 排产建议（简化贪心，只读不落库）

给定工单：剩余可派 = 上限 − dispatched；候选格 = 同车间启用产线 × [max(今日, 计划开始) .. 交期] 内工作日 × 启用班次；每格剩余产能 = 班次产能 − 已排量；按日期升序贪心填充直至覆盖剩余可派或格子用尽。返回建议列表（含 kitStatus 警示、planStartTime、planEndTime、能否按期排完标记 canFinishOnTime）。用户采纳后前端携建议参数调创建接口（is_suggest=1）。不做多工单全局优化、不考虑换型时间。

## 6. 验收标准

1. 对已下达工单执行齐套分析，kit_analysis 生成逐物料结果，工单 kit_status 回写 MAX 值；重复分析旧结果被软删不残留。
2. 欠料看板按物料汇总欠料量与影响工单数，可下钻到工单；欠料处理记录可新增、可解决。
3. 并发创建派工单不超派（FOR UPDATE + UPDATE WHERE 兜底双保险）；取消派工回退已派数量。
4. 同产线同日同班次累计排产不超产能；非工作日拒绝排产。
5. 派工状态机非法流转被拒；下发后调整必填原因并留调整日志。
6. 排产建议在交期内按工作日/班次给出可行填充，欠料工单带警示。
