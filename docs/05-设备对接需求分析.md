# 设备对接需求分析

设备对接模块用于接收现场设备产生的计数、状态和联调数据，使 MES 能够将设备产出与生产任务、工序和报工记录关联起来。

## 1. 设备计数及联调

### 功能概述

设备计数及联调用于满足现场不同设备的产量计数接入需求。羽毛球流水线中，可对植毛、注胶、烘干、分拣、包装等关键设备采集生产数量、运行状态和异常状态。

### 业务流程

1. 设备管理员维护设备接入配置，包括设备编码、采集点、计数规则和关联工序。
2. 设备完成生产动作后产生计数数据。
3. 系统接收设备计数数据，并根据设备、产线、工序和当前任务进行匹配。
4. 匹配成功后生成设备计数记录，可用于自动或半自动报工。
5. 匹配失败的数据进入异常数据池，由管理员人工处理。
6. 联调阶段记录设备通讯状态、数据格式和测试结果。

### 参与角色

- 设备管理员：维护设备接入配置和联调记录。
- 车间主管：确认设备计数是否符合生产现场。
- 操作工：核对设备计数和实际产出。
- 生产经理：查看设备计数对产量统计的影响。

### 输入输出

- 输入：设备编码、采集时间、计数值、运行状态、故障状态、关联产线、关联工序。
- 输出：设备计数记录、联调记录、自动报工数据、异常计数数据。

### 主要数据

- 设备接入配置表：记录设备、采集点、工序、数据来源和启用状态。
- 设备计数记录表：记录采集时间、计数值、增量值和匹配任务。
- 设备联调记录表：记录测试时间、测试人、通讯结果和问题说明。
- 异常数据表：记录无法匹配任务或格式异常的设备数据。

### 规则与异常

- 设备计数必须能关联到当前有效生产任务，不能关联时进入异常池。
- 重复上报的计数数据应通过采集时间、设备编码和流水号进行去重。
- 设备计数不能直接覆盖人工报工，应保留自动计数来源。
- 设备停机、维修或未绑定工序时，不应产生有效报工。
- 设备联调未通过时，不允许启用正式采集。

### 接口需求

- 接收设备计数和设备状态数据。
- 向生产报工提供设备计数依据。
- 向 OEE、电子看板和设备状态模块提供设备运行数据。
- 向异常处理人员提供无法匹配的数据清单。

### 验收标准

- 能维护设备采集配置。
- 能接收并保存设备计数数据。
- 能将设备计数关联到生产任务和工序。
- 异常数据能被识别、查询和人工处理。
- 联调记录可查询，设备接入状态清晰可见。
