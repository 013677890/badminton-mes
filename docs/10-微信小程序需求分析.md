# 微信小程序需求分析

微信小程序模块用于让管理人员和现场相关人员在移动端查看生产实时信息、生产分析和产品追溯结果。该模块以查询展示为主，不承担复杂业务维护。

## 1. 实时看板

### 功能概述

实时看板用于在微信小程序中查看当前车间生产情况，展示产量、任务进度、质量情况、异常状态和产线运行状态。

### 业务流程

1. 用户登录微信小程序。
2. 选择车间、产线、日期或班次。
3. 系统展示当前生产任务、计划数量、完成数量、达成率、不良数量和异常数量。
4. 用户可进入任务详情查看工序进度和异常信息。

### 参与角色

- 生产经理：查看整体生产情况。
- 车间主管：查看本车间任务进度。
- 产线班组长：查看本产线实时数据。

### 输入输出

- 输入：用户身份、车间、产线、日期、班次。
- 输出：实时产量、任务进度、质量指标、异常状态、设备状态摘要。

### 主要数据

- 实时生产汇总：记录计划数、完成数、达成率和不良数。
- 产线状态数据：记录运行、暂停、异常等状态。
- 异常汇总数据：记录未处理异常和超时异常。

### 规则与异常

- 所有已登录用户均可查看车间和产线数据，车间、产线仅作为筛选条件。
- 实时数据应显示最后更新时间。
- 数据源异常时，应提示数据暂不可用。

### 接口需求

- 从生产任务、生产报工、质量管理、设备管理和安灯管理读取汇总数据。

### 验收标准

- 小程序能展示车间和产线实时生产情况。
- 能查看任务进度、质量和异常摘要。
- 数据权限控制有效。

## 2. 生产分析

### 功能概述

生产分析用于在微信小程序中查看历史生产数据和质量分析，包括产量趋势、达成率、不良率、返修率和异常分布。

### 业务流程

1. 用户选择分析维度，例如日期、车间、产线、产品或班次。
2. 系统汇总历史生产和质量数据。
3. 展示趋势、排名和异常指标。
4. 用户可进入详情查看具体任务或批次。

### 参与角色

- 生产经理：查看生产趋势和管理指标。
- 车间主管：分析本车间产量和质量。
- 质检员：查看质量相关分析。

### 输入输出

- 输入：统计周期、车间、产线、产品、班次。
- 输出：产量趋势、达成率、不良率、返修率、异常次数、排名数据。

### 主要数据

- 生产统计数据：记录按周期汇总的产量和达成情况。
- 质量统计数据：记录不良、返修和检验结果。
- 异常统计数据：记录异常次数、类型和处理时长。

### 规则与异常

- 统计口径应与报表分析模块保持一致。
- 历史数据应支持按日、周、月查看。
- 空数据时应显示无数据提示。

### 接口需求

- 从报表分析模块读取统计结果。
- 所有已登录用户均可按车间、产线等维度查询。

### 验收标准

- 能按多维度查看生产分析。
- 指标与后台报表口径一致。
- 能从汇总数据进入明细。

## 3. 产品追溯

### 功能概述

小程序产品追溯用于通过扫描或输入产品批次码，查看该批次的生产过程、关键物料、质量检验、返修和异常记录。

### 业务流程

1. 用户在小程序中扫描产品批次码。
2. 系统识别批次并查询追溯链路。
3. 展示生产任务、工序报工、关键物料、检验结果、返修记录和异常记录。
4. 用户可查看追溯详情并用于现场沟通或客户问题处理。

### 参与角色

- 生产经理：查看批次完整履历。
- 质检员：追溯质量问题。
- 车间主管：追溯生产过程。

### 输入输出

- 输入：产品批次码。
- 输出：批次基本信息、工序履历、物料批次、质检记录、异常记录、返修记录。

### 主要数据

- 产品批次索引：记录批次与任务、工序和物料的关联。
- 追溯明细数据：记录报工、检验、返修和异常。

### 规则与异常

- 无效批次码应提示未查询到数据。
- 所有已登录用户均可查看追溯信息。
- 追溯结果应按业务发生时间排序。

### 接口需求

- 与产品追溯、生产报工、质量管理、返修工单和安灯管理联动。

### 验收标准

- 小程序能扫码查询产品批次。
- 能展示完整追溯链路。
- 追溯信息与后台数据一致。
