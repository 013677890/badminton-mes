# 生产组织基础资料设计

## 1. 下一切片选择

根据 [14-后端模块分工规划.md](14-后端模块分工规划.md)，A 已完成生产工单、齐套分析、派工单、
工序、工艺路线、计件工资，以及计量单位和外部工单写入。剩余标准 API 中的生产任务、设备计数和
完工单分别依赖 B/C 组尚未落地的主档；ERP 主动拉取仍缺连接地址、认证方式和字段协议，当前不适合
猜测实现。

现有派工逻辑已经依赖车间、产线、班次和工厂日历，但这些数据只有迁移种子和只读 Repository。
因此本轮先实现“车间 + 产线”基础资料管理，补齐工单与派工共同依赖的组织层级；班次和工厂日历
维护作为下一切片。

## 3. 权限

- 写操作：`ADMIN`、`PMC`；
- 查询操作：`ADMIN`、`PMC`、`WORKSHOP_MANAGER`、`CRAFT_ENGINEER`；
- Controller 仅负责参数校验、角色限制和 Service 委托。

## 4. 核心规则

### 4.1 编码、删除与并发

- 车间和产线编码限定为 ASCII 字母、数字、下划线和连字符，保存前去空格并转大写；
- 有效编码由生成列唯一索引兜底，逻辑删除后允许复用编码；
- 两类主档都使用 JPA `@Version`，修改、状态变更和删除必须携带客户端预期版本；
- 创建人、修改人和创建/更新时间完整保留。

### 4.2 车间引用规则

- 车间主管可空；填写时必须引用未删除且启用的系统用户；
- 存在启用产线、未结束工单或启用用户时，车间不能停用；
- 存在任意产线、工单、工厂日历或用户引用时，车间不能删除；
- 工单创建、修改和外部工单写入均写锁校验车间，避免与车间停用并发穿透。

### 4.3 产线引用规则

- 新建产线必须绑定未删除且启用的车间；
- 产线创建后所属车间不可修改，避免历史派工与报表口径随主档修改而漂移；
- 存在未结束派工或启用用户时，产线不能停用；
- 存在任意派工或用户引用时，产线不能删除；
- 派工创建和修改写锁校验产线，避免与产线停用并发穿透。

### 4.4 用户组织归属

系统用户创建和修改时通过 `ProductionOrganizationReferenceQuery` 校验：

- 车间、产线都为空表示不绑定生产数据范围；
- 产线非空时车间必须同时填写；
- 车间和产线必须启用，且产线必须属于所选车间；
- 已停用用户重新启用时再次执行相同校验，不能恢复到已停用的组织；
- 校验按“车间 → 产线”顺序加锁，与组织启停共用锁序。

生产模块通过 `OrganizationUserReferenceQuery` 读取用户反向引用，不跨模块直接访问系统 Repository。

## 5. V13 数据库变更

迁移文件：`backend-java/src/main/resources/db/migration/V13__production_organization_master_data.sql`。

主要变更：

- 为 `base_workshop`、`base_production_line` 增加 `create_by`、`update_by` 和 `version`；
- 将原物理编码唯一索引替换为仅约束未删除记录的生成列唯一索引；
- 增加编码前缀分页、车间状态分页和车间下产线引用校验索引；
- 不新增数据库外键，引用完整性由 Service 与数据库唯一约束共同保证。

## 6. 本轮不包含

- 班次和工厂日历维护接口；
- 产线换车间；需要调整时停用旧产线并创建新产线；
- B/C 组尚未落地的生产任务、设备计数和生产完工接口；
- ERP HTTP 客户端、认证和调度任务。

## 7. 验证重点

- 编码规范化、逻辑删除后编码复用和唯一键并发兜底；
- 客户端过期版本、停用与删除引用拦截；
- 用户车间/产线层级一致性；
- 工单、外部工单、派工与组织启停的写锁协作；
- 分页空结果短路、稳定排序和产线列表批量回填车间信息；
- V13 生成列、唯一索引、审计字段与乐观锁迁移契约。
