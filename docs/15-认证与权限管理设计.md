# 认证与权限管理设计

## 1. 定位与范围

认证与权限管理不属于 [14-后端模块分工规划.md](14-后端模块分工规划.md) 中 13 个需求模块的任何一个，是三个工作包共同依赖的公共基建。按协作边界约定"新增公共能力先沉淀到 `common`"，本模块拆成两部分：

- `com.badminton.mes.common.security`：登录上下文、拦截器、角色注解——供各模块直接使用；
- `com.badminton.mes.module.system`：用户、角色、会话的业务实现——常规三层结构。

数据表复用 [database/mes_schema.sql](database/mes_schema.sql) 已设计的 `sys_user`、`sys_role`、`sys_user_role`，不新增表。

本轮不做：菜单/按钮级权限（schema 未设计 `sys_menu`/`sys_permission`，不过度设计）、数据权限（按车间/产线过滤数据，待现场管理模块建设后按需补充）、验证码。

## 2. 技术选型

**Redis 不透明 token 会话 + Spring MVC 拦截器 + `spring-security-crypto`（仅密码哈希）。**

| 备选 | 结论 | 原因 |
| --- | --- | --- |
| 自研：Redis token + HandlerInterceptor | ✅ 采用 | 项目已有 Redis；登出/踢人/停用即下线语义天然成立；无新增重量级依赖 |
| Spring Security 7 全家桶 | 不采用 | 过滤器链与配置面对三人教学项目过重，需求只是"登录 + 角色注解" |
| Sa-Token | 不采用 | 对 Spring Boot 4 / Framework 7 的适配未经确认（STACK-003 不追未验证版本） |
| JWT | 不采用 | 无法主动失效，"停用踢人/改密下线"需另建黑名单，等于变相回到服务端会话 |

`spring-security-crypto` 是纯工具 jar（无自动配置、无过滤器链），版本由 Spring Boot BOM 仲裁，只用其 `BCryptPasswordEncoder` 满足 `sys_user.password`"加密存储"的设计（BCrypt 自带盐）。

## 3. 权限模型

粗粒度 RBAC，角色即权限。六个内置角色与 schema 注释一致，作为种子数据固化，不提供角色增删改：

| 角色编码 | 角色名称 |
| --- | --- |
| ADMIN | 管理员 |
| PMC | PMC计划员 |
| WORKSHOP_MANAGER | 车间主管 |
| TEAM_LEADER | 班组长 |
| OPERATOR | 操作工 |
| INSPECTOR | 质检员 |

访问控制两级：

1. **登录控制**：`/api/**` 默认全部要求登录，白名单仅 `POST /api/system/auth/login`。不设 `@RequiresLogin` 注解——"需要登录"是默认值而非声明项。
2. **角色控制**：`@RequiresRoles({"ADMIN", "PMC"})` 标注在 Controller 方法或类上，命中任一角色即放行；方法级注解优先于类级。

前端按 profile 返回的 `roleCodes` 控制管理员职位选项；后端注解和 Service 校验是最终防线（SEC-001）。

### 3.1 生产工单接口权限矩阵

拦截器在 token 校验通过后执行角色限制。生产工单接口按"计划职能 / 执行职能"划分：

| 接口 | ADMIN | PMC | WORKSHOP_MANAGER | 其他角色 |
| --- | :-: | :-: | :-: | :-: |
| 创建/修改/删除工单 | ✅ | ✅ | — | — |
| 下达 / 关闭 / 作废 | ✅ | ✅ | — | — |
| 暂停 / 恢复 / 完工 | ✅ | ✅ | ✅ | — |
| 详情 / 分页 / 物料需求 / 状态日志 | ✅ | ✅ | ✅ | ✅（登录即可） |

后续模块（条码、现场、质量、设备、安灯等）沿用同一机制：登录默认放行查询，写操作按职能角色用 `@RequiresRoles` 显式声明。微信小程序报表、追溯和看板查询不再按用户所属车间或产线收敛范围。

## 4. 会话设计

- 请求头 `Authorization: Bearer <token>`，token 为 `SecureRandom` 生成的 128 位随机串（hex）。
- Redis Key（`mes:system:` 前缀，与生产模块 `mes:production:` 风格一致）：

| Key | 值 | TTL | 用途 |
| --- | --- | --- | --- |
| `mes:system:login:token:{token}` | LoginUser JSON | 8h | 会话正查，拦截器命中时若剩余不足一半则滑动续期 |
| `mes:system:login:user:{userId}` | token 字符串 | 8h | 反向索引：单设备登录互踢、强制下线 |
| `mes:system:login:fail:{userNo}` | 失败计数 | 15min | 防爆破：连续 5 次失败锁定，锁定期自最后一次失败起算 |

- **单设备登录**：登录成功时若反向索引已有旧 token，先删除旧会话再写新会话。
- **强制下线**：停用、逻辑删除、重置密码、修改本人密码、调整角色，均按 userId 删除会话；用户下次请求返回 401 重新登录。
- LoginUser 载荷：`userId`、`userNo`、`userName`、`workshopId`、`lineId`、`roleCodes`。会话不回写：管理员调整用户角色时直接强制下线，重新登录后新角色生效，避免降权后旧会话继续持有原角色。

## 5. 请求处理流程

```
请求 → AuthInterceptor.preHandle
        ├─ 白名单(login) → 放行
        ├─ 无 token / Redis 无会话 → 抛 A0230 → GlobalExceptionHandler → HTTP 401
        ├─ 会话命中 → 滑动续期 → SecurityContextHolder 写入(token, LoginUser)
        ├─ 方法/类上有 @RequiresRoles 且角色无交集 → 抛 A0301 → HTTP 403
        └─ 放行 → Controller → Service 经 SecurityContextHolder 取当前操作人
     → AuthInterceptor.afterCompletion → SecurityContextHolder.clear()   ← ThreadLocal 必清理
```

`GlobalExceptionHandler` 在既有"A 类→400"映射前特判：`A0230` → 401、`A0301` → 403（API-003 四要素响应不变）。

## 6. 接口清单

### 6.1 认证 `/api/system/auth`（AuthController）

| 方法 | 路径 | 说明 | 角色 |
| --- | --- | --- | --- |
| POST | `/login` | 工号+密码登录，返回 token、用户信息、角色列表 | 匿名（唯一白名单） |
| POST | `/logout` | 删除当前会话，幂等 | 登录即可 |
| GET | `/profile` | 当前用户信息（实时读库，手机号脱敏） | 登录即可 |
| PUT | `/password` | 修改本人密码（校验旧密码），成功后强制下线 | 登录即可 |

### 6.2 用户管理 `/api/system/users`（UserController）

| 方法 | 路径 | 说明 |
| --- | --- | --- |
| POST | `/api/system/users` | 新增用户：工号唯一、初始密码必填（BCrypt 落库）、分配角色；仅 ADMIN |
| GET | `/page` | 分页查询，按工号/姓名/车间/角色/状态筛选，手机号脱敏（SEC-002）；登录即可 |
| GET | `/{id}` | 用户详情（含角色），手机号脱敏；登录即可 |
| PUT | `/{id}/assignment` | 分配职位、车间和产线；登录即可，只有 ADMIN 可授予或撤销 ADMIN |
| PUT | `/{id}` | 修改姓名/手机/车间/产线/角色（工号、密码不在此修改）；仅 ADMIN |
| PUT | `/{id}/status` | 启用/停用；停用即强制下线；不能停用自己；仅 ADMIN |
| PUT | `/{id}/password/reset` | 管理员重置密码，重置后强制下线；仅 ADMIN |
| DELETE | `/{id}` | 逻辑删除；不能删除自己；删除后强制下线；仅 ADMIN |

### 6.3 角色 `/api/system/roles`（RoleController）

| 方法 | 路径 | 说明 | 角色 |
| --- | --- | --- | --- |
| GET | `/api/system/roles` | 启用角色列表（用户表单下拉） | 登录即可 |
| GET | `/{id}/users` | 按角色反查启用用户（对应 `idx_role_id` 设计意图：安灯按角色匹配处理人，供 C 组复用） | 登录即可 |

### 6.4 内部能力（非 HTTP）

- `SecurityContextHolder.getRequiredLoginUser()` / `getRequiredLoginUserId()`：各模块 Service 取当前操作人，替换 `WorkOrderServiceImpl` 中的 `DEFAULT_OPERATOR_ID` 占位。
- `LoginSessionReader`（common 接口，system 模块实现）：拦截器读会话，避免 common 反向依赖 module。

## 7. 错误码

沿用宏观错误码复用策略（ERRCODE-006/008，附表 3）。

**`GlobalErrorCodeConstants` 新增（跨模块通用）：**

| 码 | 含义 | HTTP |
| --- | --- | --- |
| A0230 | 用户登录已过期（未登录/token 失效） | 401 |
| A0301 | 访问未授权（角色不符） | 403 |

**`SystemErrorCodeConstants`（system 模块内）：**

| 码 | 场景 |
| --- | --- |
| A0200 | 工号或密码错误（账户不存在与密码错误统一提示，防撞库枚举账号；服务端日志区分） |
| A0211 | 密码连续错误次数超限，锁定中 |
| A0202 | 账户已停用 |
| A0201 | 用户不存在（用户管理接口） |
| A0506 | 工号已存在 |
| A0210 | 旧密码错误（修改本人密码） |
| A0402 | 初始密码缺失 / 所选角色不可用 |
| A0440 | 不能停用或删除当前登录账号 |

## 8. 数据与迁移

Flyway `V3__system_auth.sql`：

- 照抄 schema 建 `sys_user`、`sys_role`、`sys_user_role`；
- 种子数据：六个角色 + `admin` 账号（初始密码 `admin123`，BCrypt 哈希落库，首次登录后应修改）+ admin↔ADMIN 关联。

`workshop_id`/`line_id` 的档案有效性校验暂不做（基础资料查询服务由各模块后续提供，避免 system 跨包访问 production 的 repository），仅存值。

## 10. 验收标准

1. 未携带 token 访问任意业务接口返回 401 + A0230；角色不符返回 403 + A0301。
2. 所有登录用户可查询人员并分配普通职位；只有 ADMIN 可授予或撤销 ADMIN，账号新增、删除、停用和密码重置仍仅 ADMIN 可用。
3. 连续 5 次密码错误后登录被锁定 15 分钟；正确密码在锁定期内同样被拒。
4. 停用/删除/重置密码后，该用户既有会话立即失效。
5. 工单操作留痕（`create_by`、`operate_by`）记录真实登录用户，不再是占位值 1。
6. 密码只存 BCrypt 哈希；接口不返回密码字段；手机号脱敏展示。
