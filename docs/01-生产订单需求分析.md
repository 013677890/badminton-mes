# 生产订单需求分析

生产订单模块用于接收和管理羽毛球生产指令，是 MES 生产业务的起点。该模块主要包括生产工单、齐套分析与欠料看板、派工单三个功能。

## 1. 生产工单

### 功能概述

生产工单用于记录工厂下达给车间的生产指令，明确生产产品、计划数量、交期、生产批次、BOM 版本和目标车间。羽毛球场景中，生产工单可对应某一型号羽毛球的生产批次，例如训练球、比赛球或指定客户订单产品。

### 业务流程

1. PMC 计划员通过 ERP 同步、Excel 导入或手工录入创建生产工单。
2. 系统校验产品、BOM、工艺路线、计划数量和交期是否完整。
3. 车间主管确认工单可执行后，将工单拆分或转换为生产任务单。
4. 工单在执行过程中汇总投入数量、完工数量、不良数量和返修数量。
5. 工单全部任务完成后，系统将状态更新为已完工或待关闭。

### 参与角色

- PMC 计划员：创建、导入、修改和下达生产工单。
- 车间主管：接收工单，确认生产条件，跟踪执行状态。
- 生产经理：查看工单进度和达成情况。
- ERP 系统：提供外部工单来源。

### 输入输出

- 输入：工单号、产品编码、产品名称、规格型号、计划数量、计划开始时间、计划完成时间、客户信息、BOM 版本、工艺路线、优先级。
- 输出：生产工单列表、工单详情、工单状态、工单进度、工单异常提示。

### 主要数据

- 生产工单主表：记录工单基本信息、计划信息和状态。
- 工单产品明细：记录产品型号、单位、计划数量和批次信息。
- 工单物料需求：记录球头、羽片、胶水、线圈、包装材料等需求数量。
- 工单状态日志：记录创建、下达、暂停、关闭等状态变化。

### 规则与异常

- 工单号必须唯一，不允许重复。
- 未维护 BOM 或工艺路线的产品不允许直接下达生产。
- 工单已下达后，如需修改计划数量或交期，应记录变更原因。
- 工单执行数量不能超过允许的超产比例。
- 已关闭工单不能继续生成生产任务单。

### 接口需求

- 支持从 ERP 读取生产工单。
- 支持向报表模块提供工单计划数、完成数和进度数据。
- 支持向现场管理模块提供可排产工单。

### 验收标准

- 能新增、导入、查询、修改、下达和关闭生产工单。
- 能查看工单计划数量、已派工数量、已完工数量和不良数量。
- 缺少 BOM 或工艺路线时，系统能给出明确提示。
- 工单状态变化有日志可查。

## 2. 齐套分析与欠料看板

### 功能概述

齐套分析用于判断生产任务所需物料是否满足开工条件，欠料看板用于展示缺料物料、缺料数量、影响工单和预计到料情况。齐套分析以生产工单为对象进行计算，齐套及欠料结果面向产线任务发布，供排产和开工决策使用。羽毛球生产中重点分析球头、羽片、胶水、包装材料等关键物料。

### 业务流程

1. 系统根据生产工单计划数量和 BOM 计算物料需求。
2. 系统读取可用库存、已锁定库存和在途物料信息。
3. 生成齐套结果，标识齐套、部分齐套或欠料。
4. PMC 或仓库人员根据欠料看板协调采购、领料或调整排产。
5. 欠料补齐后，系统更新齐套状态。

### 参与角色

- PMC 计划员：查看齐套结果并调整生产优先级。
- 仓库人员：确认库存和物料可用情况。
- 车间主管：根据齐套情况决定是否开工。
- 生产经理：查看影响交期的欠料风险。

### 输入输出

- 输入：生产工单、BOM、库存数量、已占用数量、在途数量、计划开工日期。
- 输出：齐套状态、欠料清单、欠料数量、影响工单、建议处理方式。

### 主要数据

- 物料需求明细：记录每个工单所需物料和标准用量。
- 库存可用量：记录可用、锁定、在检、在途数量。
- 齐套分析结果：记录齐套状态、缺料数量和分析时间。
- 欠料处理记录：记录责任人、预计到料时间和处理说明。

### 规则与异常

- 可用库存应扣除已锁定库存和质检未放行库存。
- 同一物料被多个工单占用时，应按优先级或交期分配。
- 欠料数量大于零时，相关工单不能直接标记为齐套。
- 库存数据异常或物料编码缺失时，应提示人工确认。

### 接口需求

- 从 WMS 或库存模块读取物料可用量。
- 从 ERP 或采购系统读取在途物料信息。
- 向生产订单和现场管理模块返回齐套状态。

### 验收标准

- 能按工单生成物料需求和齐套结果。
- 能清晰展示欠料物料、欠料数量和影响工单。
- 库存变化后能重新分析齐套状态。
- 欠料处理过程有记录可查。

## 3. 派工单

### 功能概述

派工单用于根据工单交期、产线产能、工厂日历和生产优先级，将生产任务安排到具体产线和班次。系统可给出排产建议，也支持人工调整。

### 业务流程

1. PMC 或车间主管选择待排产生产工单。
2. 系统根据交期、产线能力、班次和齐套状态生成派工建议。
3. 车间主管调整产线、开始时间、结束时间和计划数量。
4. 审核后生成派工单，并下发到现场管理模块。
5. 派工单执行过程中同步生产进度和异常状态。

### 参与角色

- PMC 计划员：制定生产计划和派工建议。
- 车间主管：确认派工单并下发产线。
- 产线班组长：接收派工单并组织生产。
- 生产经理：查看产线负荷和计划达成。

### 输入输出

- 输入：生产工单、齐套状态、产线能力、班次、工厂日历、交期、优先级。
- 输出：派工单、产线排程、班次任务、派工状态。

### 主要数据

- 派工单主表：记录派工单号、来源工单、产线、班次、计划数量和状态。
- 派工明细：记录产品、工序、计划开始时间和计划结束时间。
- 产线能力数据：记录产线可生产产品、标准产能和可用时间。
- 排产日志：记录建议排产和人工调整过程。

### 规则与异常

- 未齐套工单可生成待确认派工，但不建议直接开工。
- 同一产线同一时间段不能重复安排超出产能的任务。
- 派工数量不能超过工单未派数量。
- 派工单下发后如需调整，应记录调整原因。

### 接口需求

- 向现场管理模块下发生产任务。
- 从工艺管理模块读取工艺路线。
- 从设备管理模块读取产线可用状态。
- 向报表和看板模块提供排产计划数据。

### 验收标准

- 能按工单生成派工单。
- 能根据产线、班次和日期查看排产结果。
- 能修改、审核、下发和取消派工单。
- 派工单下发后能在现场端查看并执行。
