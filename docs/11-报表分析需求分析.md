# 报表分析需求分析

报表分析模块用于对生产、质量、物料和车间运行数据进行汇总查询，为管理决策和问题追溯提供依据。

## 1. 关键物料追溯

### 功能概述

关键物料追溯用于通过制令单号（即生产工单号）、产品批次或物料批次查询产品与关键物料之间的关系，可查询制令单的产品信息、同批产品信息、同批次物料信息，以及使用同批次物料的其他产品信息。羽毛球生产中关键物料包括球头、羽片、胶水、线圈和包装材料等。

### 业务流程

1. 用户输入生产工单、产品批次或物料批次。
2. 系统查询该工单使用的物料批次。
3. 系统展示同批物料被哪些产品批次使用。
4. 发生质量问题时，可根据物料批次反查受影响产品。

### 参与角色

- 质检员：追溯质量问题来源。
- 车间主管：定位受影响生产批次。
- 生产经理：查看物料风险范围。

### 输入输出

- 输入：制令单号（工单号）、产品批次、物料编码、物料批次。
- 输出：产品信息、同批产品、同批物料、受影响批次。

### 主要数据

- 关键物料使用记录：记录产品批次与物料批次关系。
- 工单物料明细：记录工单对应物料需求和实际使用。
- 追溯查询记录：记录查询条件和结果。

### 规则与异常

- 关键工序应强制记录物料批次。
- 物料批次缺失时，追溯结果应提示数据不完整。
- 查询结果应支持正向追溯和反向追溯。

### 接口需求

- 从生产报工、关键物料报工、生产工单和库存业务读取数据。

### 验收标准

- 能按产品批次查询关键物料批次。
- 能按物料批次反查受影响产品。
- 追溯链路清晰且可导出。

## 2. 产品追溯

### 功能概述

产品追溯用于通过产品批次码查询生产全过程，覆盖生产任务、工序报工、设备、人员、质量、返修和异常记录。

### 业务流程

1. 用户输入或扫描产品批次码。
2. 系统查询批次对应的生产任务和工艺路线。
3. 系统按时间顺序展示工序履历。
4. 同步展示质量检验、返修和异常处理记录。

### 参与角色

- 质检员：追溯不良批次。
- 车间主管：追溯生产过程。
- 生产经理：查看批次完整履历。

### 输入输出

- 输入：产品批次码、工单号、任务单号。
- 输出：批次信息、工序履历、报工记录、质检结果、异常记录、返修记录。

### 主要数据

- 产品追溯主索引：记录批次与各业务单据关系。
- 工序履历表：记录批次经过的工序、人员和设备。
- 质量追溯表：记录检验和不良处理结果。

### 规则与异常

- 批次码是追溯主键，必须唯一。
- 追溯数据应按业务时间排序。
- 缺失环节应明确提示，不能隐藏数据断点。

### 接口需求

- 与生产任务、生产报工、质量管理、设备管理、返修和安灯管理联动。

### 验收标准

- 能按批次码查询完整生产履历。
- 能展示质量、返修和异常记录。
- 查询结果可用于问题定位。

## 3. 产量报表查询

### 功能概述

产量报表查询用于按不同时间和维度统计生产产量，支持查看产量趋势和汇总结果。

### 业务流程

1. 用户选择统计周期、车间、产线、产品或班次。
2. 系统汇总生产报工和完工数据。
3. 展示计划数量、实际产量、良品数、不良数和达成率。
4. 用户可导出报表或查看明细。

### 参与角色

- 车间主管：查看本车间产量。
- 生产经理：查看全厂产量趋势。
- PMC 计划员：对比计划与实际。

### 输入输出

- 输入：日期范围、车间、产线、产品、班次、工单。
- 输出：产量汇总、产量趋势、达成率、不良数量、明细数据。

### 主要数据

- 报工汇总数据：记录工序和任务产量。
- 完工统计数据：记录完工数量和批次。
- 产量报表缓存：记录按周期汇总结果。

### 规则与异常

- 产量统计口径应明确区分报工产量和完工产量。
- 同一报工记录不能重复统计。
- 统计结果应支持按日、周、月汇总。

### 接口需求

- 从生产报工和生产完工单读取产量数据。
- 向小程序和电子看板提供汇总指标。

### 验收标准

- 能按多个维度查询产量。
- 能展示计划与实际差异。
- 报表数据可导出。

## 4. 生产实时信息

### 功能概述

生产实时信息用于查询当前正在生产的工单、任务和产线状态，包括投入数量、产出数量、不良数量和任务进度。

### 业务流程

1. 用户进入生产实时信息页面。
2. 系统自动加载当前进行中的工单和任务。
3. 展示各产线计划、完成、不良、异常和状态。
4. 用户可查看单个任务的工序明细。

### 参与角色

- 车间主管：监控现场生产。
- 生产经理：查看全厂实时状态。
- 产线班组长：查看本产线进度。

### 输入输出

- 输入：车间、产线、当前日期、班次。
- 输出：在制工单、任务进度、投入数量、产出数量、不良数量、异常状态。

### 主要数据

- 在制任务视图：记录当前执行中的任务。
- 实时进度数据：汇总报工、设备计数和异常。

### 规则与异常

- 只显示未完工、未关闭的生产任务。
- 数据应显示最后刷新时间。
- 异常任务应突出显示。

### 接口需求

- 从生产任务、生产报工、设备状态和安灯管理读取数据。

### 验收标准

- 能实时查看在产任务。
- 能查看投入、产出和不良数量。
- 异常任务能被快速识别。

## 5. 不良查询

### 功能概述

不良查询用于按日期、班组、工序、不良现象和不良点位查询不良记录，并进行汇总分析。

### 业务流程

1. 用户设置查询条件。
2. 系统从报工和质检记录中汇总不良数据。
3. 展示不良数量、不良率、不良原因排名和趋势。
4. 用户进入明细查看具体批次和处理结果。

### 参与角色

- 质检员：分析质量问题。
- 车间主管：定位生产过程问题。
- 生产经理：查看质量改善效果。

### 输入输出

- 输入：日期、班组、产线、工序、不良现象、不良点位、产品。
- 输出：不良明细、不良汇总、不良率、原因排名、趋势数据。

### 主要数据

- 不良报工明细：记录生产报工中的不良。
- 检验不良记录：记录质检发现的不良。
- 不良统计表：记录按维度汇总结果。

### 规则与异常

- 不良数量应能追溯到来源报工或检验单。
- 同一不良不能重复统计。
- 不良原因为空的数据应单独列为未分类。

### 接口需求

- 从生产报工、质量管理和返修工单读取不良数据。

### 验收标准

- 能按多条件查询不良。
- 能查看不良原因和趋势。
- 能从汇总进入明细。

## 6. 车间生产时段报表

### 功能概述

车间生产时段报表用于按日、周、月查询各车间生产情况，展示产量、达成率、质量和异常等指标。

### 业务流程

1. 用户选择统计时段和车间。
2. 系统汇总该时段内生产任务、报工、完工、质检和异常数据。
3. 展示车间产量、达成率、不良率、异常次数和停线时长。
4. 用户可对比不同车间或不同时段。

### 参与角色

- 生产经理：查看车间绩效。
- 车间主管：查看本车间表现。
- PMC 计划员：分析计划达成情况。

### 输入输出

- 输入：统计周期、车间、产品、班次。
- 输出：车间产量、计划达成率、不良率、异常次数、停线时长。

### 主要数据

- 车间生产汇总表：记录周期性汇总指标。
- 车间质量汇总表：记录不良和检验结果。
- 车间异常汇总表：记录异常和停线影响。

### 规则与异常

- 日、周、月统计口径应保持一致。
- 停线时长应来自异常或设备状态记录。
- 指标为空时应显示为零或无数据，避免误判。

### 接口需求

- 从生产任务、报工、完工、质量、设备和异常模块读取数据。

### 验收标准

- 能按日、周、月查询车间生产情况。
- 能对比产量、质量和异常指标。
- 报表结果可导出。
