# 质量管理需求分析

质量管理模块用于维护检验项目、检验方案，并对生产过程和成品流转进行质量检验。羽毛球场景中重点关注外观、重量、羽片排列、球头牢固度、飞行稳定性等质量指标。

## 1. 检验项目分类

### 功能概述

检验项目分类用于维护质量检验项目的分类和具体检测项目，便于在不同检验方案中复用。

### 业务流程

1. 质检员维护检验分类。
2. 在分类下维护检验项目、标准值、上限、下限和判定方式。
3. 检验标准方案引用检验项目。
4. 检验单按项目录入实测值和判定结果。

### 参与角色

- 质检员：维护和使用检验项目。
- 工艺工程师：确认工序相关质量标准。
- 生产经理：查看质量指标统计。

### 输入输出

- 输入：分类名称、检验项目、标准值、上限、下限、检验方法、是否必检。
- 输出：检验项目库、项目分类、可选检验项。

### 主要数据

- 检验项目分类表：记录分类编码、名称和状态。
- 检验项目表：记录项目名称、单位、标准、上下限和判定方式。
- 检验方法表：记录检验方法和说明。

### 规则与异常

- 检验项目编码必须唯一。
- 已被方案引用的检验项目不能直接删除。
- 数值型项目应设置单位和上下限。
- 必检项目在检验单中不能为空。

### 接口需求

- 向检验标准方案和各类检验单提供检验项目。

### 验收标准

- 能维护检验分类和检验项目。
- 检验项目可被检验方案引用。
- 必检、数值范围和判定方式能正确保存。

## 2. 检验标准方案

### 功能概述

检验标准方案用于为不同产品、客户或检验场景定义检验项目组合和判定标准。羽毛球可按产品等级或客户要求设置不同检验方案。

### 业务流程

1. 质检员创建检验方案。
2. 选择适用产品、客户、检验类型和生效日期。
3. 配置检验项目、抽样数量、判定标准和是否必检。
4. 审核后方案生效。
5. 检验单生成时自动带出适用方案。

### 参与角色

- 质检员：维护检验方案。
- 质量主管：审核检验方案。
- 车间主管：查看生产相关检验要求。

### 输入输出

- 输入：方案名称、适用产品、客户、检验类型、检验项目、抽样规则、判定标准。
- 输出：检验方案、方案版本、检验项目清单。

### 主要数据

- 检验方案主表：记录方案名称、适用范围、版本和状态。
- 检验方案明细：记录检验项目、标准和抽样要求。
- 方案版本表：记录方案变更和生效历史。

### 规则与异常

- 同一产品同一检验类型只能有一个默认生效方案。
- 方案变更应形成新版本。
- 未审核方案不能用于正式检验。
- 已用于检验单的方案版本不能直接修改历史结果。

### 接口需求

- 向首末件检验、巡检、成品入库检验和成品发货检验提供标准方案。

### 验收标准

- 能创建、审核、停用检验方案。
- 检验单能自动带出适用方案。
- 方案版本可追溯。

## 3. 首末件检验单

### 功能概述

首末件检验单用于记录产品开始生产时的首件检验和生产结束前的末件检验，确保批量生产前后质量稳定。

### 业务流程

1. 生产任务开工后，系统触发首件检验要求。
2. 质检员按检验方案录入首件检验数据。
3. 首件合格后，产线允许继续批量生产。
4. 任务结束前，质检员执行末件检验。
5. 检验结果与生产任务和产品批次关联。

### 参与角色

- 质检员：执行首末件检验。
- 产线班组长：配合送检和处理异常。
- 车间主管：确认不合格后的处理措施。

### 输入输出

- 输入：生产任务、产品批次、检验方案、实测值、判定结果、不合格说明。
- 输出：首件检验单、末件检验单、检验结论、放行或停线建议。

### 主要数据

- 首末件检验单主表：记录任务、产品、检验类型和结论。
- 检验项目结果表：记录各项目实测值和判定。
- 检验异常处理表：记录不合格处理意见。

### 规则与异常

- 启用首检规则时，首检未通过不能批量报工。
- 检验项目缺失时不能提交检验单。
- 首件不合格应触发异常或返工处理。
- 末件不合格应提示追溯本批次过程数据。

### 接口需求

- 从生产任务和检验方案读取基础数据。
- 向生产报工、安灯管理、产品追溯和报表提供检验结果。

### 验收标准

- 能生成并填写首末件检验单。
- 首检不合格时能限制后续批量生产。
- 检验结果能关联到产品批次。

## 4. 巡检单

### 功能概述

巡检单用于记录生产过程中质检员对产线产品进行定时或按批次巡检的结果。

### 业务流程

1. 系统根据生产任务和巡检频率生成巡检任务。
2. 质检员到产线抽样检查。
3. 录入检验项目实测值、不良现象和判定结果。
4. 不合格时通知班组长处理，并可发起安灯异常。
5. 巡检结果进入质量报表和产品追溯。

### 参与角色

- 质检员：执行巡检。
- 产线班组长：处理巡检异常。
- 车间主管：跟踪质量问题。

### 输入输出

- 输入：生产任务、巡检时间、抽样数量、检验项目、实测值、不良信息。
- 输出：巡检单、巡检结论、异常通知、质量统计数据。

### 主要数据

- 巡检单主表：记录任务、产线、批次、巡检时间和结论。
- 巡检项目结果表：记录各项目结果。
- 巡检异常表：记录不合格项目和处理结果。

### 规则与异常

- 巡检频率应按产品或产线规则生成。
- 不合格项目必须填写原因和处理建议。
- 逾期未巡检应提醒质检员。
- 严重不合格可触发暂停生产。

### 接口需求

- 与生产任务、检验方案、安灯管理、产品追溯和报表分析联动。

### 验收标准

- 能生成和填写巡检单。
- 能记录巡检不合格和处理结果。
- 巡检结果能用于质量统计。

## 5. 成品入库检验单

### 功能概述

成品入库检验单用于产品生产完成后，在入库前对成品进行抽检，防止不合格品进入库存。

### 业务流程

1. 生产完工后生成待入库检验任务。
2. 质检员按检验方案抽检成品批次。
3. 录入检验结果并给出合格、让步接收、返修或报废结论。
4. 合格批次允许办理入库。
5. 不合格批次进入返修或异常处理流程。

### 参与角色

- 质检员：执行入库检验。
- 仓库人员：根据检验结论办理入库。
- 车间主管：处理不合格批次。

### 输入输出

- 输入：生产完工单、产品批次、抽样数量、检验项目、实测值。
- 输出：成品入库检验单、检验结论、入库放行状态。

### 主要数据

- 入库检验单主表：记录完工单、批次和结论。
- 入库检验明细：记录项目结果和不良信息。
- 放行记录表：记录入库放行状态。

### 规则与异常

- 未完成入库检验的批次不能作为合格品入库。
- 不合格批次应明确返修、报废或让步接收。
- 让步接收需要权限和原因记录。

### 接口需求

- 从生产完工单读取待检批次。
- 向 WMS 或入库业务提供放行结果。
- 向产品追溯和质量报表提供检验数据。

### 验收标准

- 能根据完工单生成入库检验单。
- 检验合格后能形成入库放行状态。
- 不合格处理结果可追溯。

## 6. 成品发货检验单

### 功能概述

成品发货检验单用于在成品销售发货前，对库存批次进行抽检并形成出厂检验记录，防止质量问题流向客户。

### 业务流程

1. 根据销售发货通知或发货批次生成发货检验任务。
2. 质检员对待发货批次进行抽检。
3. 录入外观、数量、包装、标签和关键质量项目。
4. 检验合格后允许发货。
5. 不合格时冻结发货批次并通知相关人员处理。

### 参与角色

- 质检员：执行发货检验。
- 仓库人员：配合抽检和发货。
- 销售或生产经理：查看出厂质量情况。

### 输入输出

- 输入：发货通知、产品批次、客户、抽样数量、检验项目。
- 输出：发货检验单、检验结论、出厂检验报告、发货放行状态。

### 主要数据

- 发货检验单主表：记录客户、批次、检验状态和结论。
- 发货检验明细：记录项目结果和不良信息。
- 出厂检验报告：记录可对外输出的检验结论。

### 规则与异常

- 未通过发货检验的批次不能发货。
- 客户有专用检验方案时，应优先使用客户方案。
- 发货检验不合格应冻结对应批次。

### 接口需求

- 从 WMS 或销售发货通知获取待检批次。
- 向发货业务返回放行或冻结结果。
- 向产品追溯提供出厂检验信息。

### 验收标准

- 能生成和填写发货检验单。
- 能输出出厂检验结论。
- 不合格批次能被限制发货。
